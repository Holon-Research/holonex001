PRD — Auto-Iterative Dharma Reasoning

Repo context: Next.js + Vercel AI SDK with visible reasoning. We’ll extend the server route to auto-iterate until a stopping condition based on the Dharma score is met.

Objective

Each reasoning step is rendered on the frontend as it streams, with a structured Thought + Dharma score. The server automatically triggers the next step until the score converges/improves sufficiently, then emits a final answer.

Contract for each reasoning step (unchanged)

parameters: z.object({
  title: z.string().describe("The title of this reasoning step"),
  content: z.string().describe(
    "The content of the reasoning step. WRITE OUT ALL OF YOUR WORK. Include mathematical proof where relevant."
  ),
  dharma: dharmaSchema.describe(
    "Evaluate this reasoning step against the four contemplative principles"
  ),
  nextStep: z.enum(["continue", "finalAnswer"]).describe(
    "Whether to continue reasoning or provide final answer"
  ),
})

dharmaSchema (shared)

const dharmaSchema = z.object({
  mindfulness: z.number().min(0).max(1),
  emptiness: z.number().min(0).max(1),
  nonDuality: z.number().min(0).max(1),
  boundlessCare: z.number().min(0).max(1),
  rationale: z.string(),
});

Termination policy (server-side)

  • Score aggregation: dharmaScore = mean(mindfulness, emptiness, nonDuality, boundlessCare).
  • Stop when any of the following:
    • Target threshold met: dharmaScore ≥ DHARMA_TARGET (default 0.9).
    • Convergence: improvement < EPSILON (default 0.02) for PATIENCE consecutive steps (default 2).
    • Max steps: MAX_STEPS (default 10) reached.
    • Model explicitly returns nextStep:"finalAnswer".

Config via env:

DHARMA_TARGET=0.9
DHARMA_EPSILON=0.02
DHARMA_PATIENCE=2
DHARMA_MAX_STEPS=6

Server loop (concept)

  • Route: app/api/reason/route.ts
  • After each streamed step:
    1. Score with scoreDharma(step.content).
    1. Emit {title, content, dharma, nextStep} immediately (SSE chunk).
    1. Update convergence state and decide to continue/stop.
    1. If continuing, append the step (and score) to the running context and prompt the model to refine toward higher Dharma score.

Pseudocode

let best = 0, patience = 0;
for (let i = 1; i <= MAX_STEPS; i++) {
  const step = await generateStep(context);         // via Vercel AI SDK
  const dharma = scoreDharma(step.content);         // returns object + mean
  const score = mean(dharma);
  emit({ ...step, dharma, nextStep: "continue" });  // stream to UI now

  if (score > best + EPSILON) { best = score; patience = 0; } else { patience++; }

  const stop =
    score >= TARGET || patience >= PATIENCE || i === MAX_STEPS || step.nextStep === "finalAnswer";

  if (stop) {
    emit({ ...finalize(context, step), dharma, nextStep: "finalAnswer" });
    break;
  }

  context = refineContext(context, step, dharma);   // include Thought+score and a refinement instruction
}

Frontend UX

  • Reasoning Panel streams cards: Title, Thought (content), Dharma breakdown (bars + rationale), running score.
  • A subtle “Improving…” ticker shows progress toward target; then a Final Answer appears as Text.
  • Timeline is scrollable.

Data & Telemetry

  • Per step log: { runId, stepIndex, title, content, dharma, score, tokensIn/out, latencyMs }
  • Storage: start with IndexedDB (client) or JSON file (server); add Supabase DB later.
  • Flags: LOG_STEPS=true to persist; redact secrets.

lib/dharma.ts

import { dharmaSchema } from "./schemas";

export function scoreDharma(text: string) {
  const clamp = (n: number) => Math.max(0, Math.min(1, n));
  // v0 heuristic signals; swap later with a learned scorer
  const mindfulness  = clamp(text.match(/assumption|bias|check/i) ? 0.7 : 0.4);
  const emptiness    = clamp(text.match(/uncertain|could be wrong|contingent/i) ? 0.7 : 0.3);
  const nonDuality   = clamp(text.match(/both\/and|complementary|trade[- ]off/i) ? 0.6 : 0.3);
  const boundlessCare= clamp(text.match(/stakeholder|harm|accessibility|ecosystem/i) ? 0.6 : 0.3);
  const rationale = "Heuristic v0 based on reflective phrases and multi-perspective markers.";
  return dharmaSchema.parse({ mindfulness, emptiness, nonDuality, boundlessCare, rationale });
}

export function mean(d: {mindfulness:number;emptiness:number;nonDuality:number;boundlessCare:number}) {
  return (d.mindfulness + d.emptiness + d.nonDuality + d.boundlessCare) / 4;
}

app/api/reason/route.ts (concept)

import { streamText } from "ai";
import { NextRequest } from "next/server";
import { scoreDharma, mean } from "@/lib/dharma";

const TARGET  = Number(process.env.DHARMA_TARGET ?? 0.75);
const EPSILON = Number(process.env.DHARMA_EPSILON ?? 0.02);
const PATIENCE= Number(process.env.DHARMA_PATIENCE ?? 2);
const MAX_STEPS = Number(process.env.DHARMA_MAX_STEPS ?? 6);

export async function POST(req: NextRequest) {
  const { prompt, provider } = await req.json();
  let best = 0, patience = 0, stepIndex = 0;
  let context = seedSystemPrompt(prompt); // includes instructions to emit {title,content,nextStep}

  const stream = new ReadableStream({
    async start(controller) {
      while (stepIndex < MAX_STEPS) {
        stepIndex++;
        const step = await generateStep(provider, context); // one structured step
        const dharma = scoreDharma(step.content);
        const score = mean(dharma);

        controller.enqueue(encode({ type:"step", data:{ ...step, dharma, score }}));

        if (score > best + EPSILON) { best = score; patience = 0; } else { patience++; }

        const stop = score >= TARGET || patience >= PATIENCE || step.nextStep === "finalAnswer";
        if (stop) {
          const final = await finalizeAnswer(provider, context, step);
          controller.enqueue(encode({ type:"final", data:{ ...final, dharma, score } }));
          break;
        }

        context = refineContext(context, step, dharma, score, best); // instruct to improve score
      }
      controller.close();
    }
  });

  return new Response(stream, { headers: { "Content-Type": "text/event-stream" }});
}

components/ReasoningPanel.tsx (behavior)

Risks & mitigations

  • Goodhart on score: keep heuristic transparent; add parsimony hints; later add verifier head.
  • Runaway loops: MAX_STEPS + PATIENCE cap; telemetry alerts when plateau detected.
  • Malformed steps: Zod validation + retry with stricter instructions.
